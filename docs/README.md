# docs/

Astro Starlight source for the AIMS OS documentation site, deployed at
https://a-i-m-s-senegal.github.io/aims-os/docs/.

## Local dev

```bash
cd docs
npm install
npm run dev      # http://localhost:4321/aims-os/docs/
```

## Build

```bash
npm run build    # outputs to docs/dist/
```

The CI workflow `.github/workflows/publish-apt-repo.yml` runs the
same `npm install && npm run build` on every dispatch and merges
the built tree into `apt-repo/site/docs/` before deploying both
to GitHub Pages in a single artefact.

## Why no lockfile yet

`package-lock.json` is gitignored for now to keep this directory
copy-paste portable across machines that aren't all on the same npm
version. Once the docs structure stabilises, commit a lockfile and
switch the CI step from `npm install` to `npm ci`.

## Adding a page

1. Create `src/content/docs/<section>/<slug>.md(x)` with frontmatter
   `title:` and (optional) `description:`.
2. Either:
   - List it explicitly in the `sidebar` array of `astro.config.mjs`
     so it appears in a curated position
   - Or place it under a directory listed with `autogenerate:
     { directory: '<section>' }` (currently `troubleshooting/` and
     `contributing/`)
3. Rebuild: `npm run build` locally or push and let CI redeploy.
