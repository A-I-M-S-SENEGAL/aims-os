AIMS OS apt repository

Official apt repo for the AIMS OS metapackages, served from GitHub Pages. Install the AIMS OS scientific stack on top of any Debian Trixie box, without flashing the full ISO.

One-time setup

Trust the signing key, add the deb822 source, refresh apt.

sudo curl -fsSL https://a-i-m-s-senegal.github.io/aims-os/aims-os-archive-keyring.gpg \
    -o /usr/share/keyrings/aims-os-archive-keyring.gpg

sudo tee /etc/apt/sources.list.d/aims-os.sources >/dev/null <<'EOF'
Types: deb
URIs: https://a-i-m-s-senegal.github.io/aims-os
Suites: trixie
Components: main
Architectures: amd64 arm64
Signed-By: /usr/share/keyrings/aims-os-archive-keyring.gpg
EOF

sudo apt update

Pick what you need

Filière / layerCommand
Regular (Mathematical Sciences)sudo apt install aims-os-math
Coop Big Data Analyticssudo apt install aims-os-bigdata
Coop Computer Securitysudo apt install aims-os-security
GNOME desktop layersudo apt install aims-os-desktop
System baseline (firmware, locales, CLI)sudo apt install aims-os-core
Everythingsudo apt install aims-os-{core,desktop,math,bigdata,security}

Both aims-os-bigdata and aims-os-security depend on aims-os-math, so installing either pulls the SciPy / R / LaTeX / Jupyter scientific baseline with them.

Verify the signature

gpg --show-keys /usr/share/keyrings/aims-os-archive-keyring.gpg

Expected fingerprint:

7775 7473 70C3 E86F A12D  06D7 CEAB 168E 6D2E 30FF

What's in each metapackage

The debian/control files in the source repo are the truth. Every metapackage lists exactly what it pulls in its Depends: field.

Branding package note

aims-os-branding (wallpapers, GRUB theme, Plymouth splash, Calamares branding) ships only with the AIMS OS ISO. The standalone apt repo skips it because (a) the assets are generated by an ImageMagick / librsvg / optipng pipeline that does not belong in CI, and (b) replacing the wallpapers on someone's existing Debian without warning is rude. Grab the ISO if you want the full branded experience.
